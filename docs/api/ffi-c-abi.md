# FFI / C ABI

The safe C ABI of the `carapace-ffi` crate — for embedding Carapace into a native host (Swift, Flutter, C/C++). This is **ABI 3.2**: a carapace-owned render thread with a command queue, an IOSurface frame pool, and seamless skin swapping. Apple-only today.

Crate: `crates/carapace-ffi/`. Canonical header: `crates/carapace-ffi/include/carapace.h` (generated by cbindgen, kept fresh by a CI test — `tests/header.rs`). Reflects source as of 2026-07-04.

> **Platform:** macOS/iOS only. On other targets the crate builds as an empty shell (so Linux CI stays green) but ships no functionality. Windows (D3D11 shared handle), Linux (Vulkan dmabuf), and Android (AHardwareBuffer) are designed-but-deferred. The two non-Apple-gated exports are `carapace_abi_version` and `carapace_last_error`.

## Model in one paragraph

The host builds a `CarapaceCreateDesc` (skin path, a callback vtable, a pool of BGRA IOSurfaces, dimensions) and calls `carapace_create`, which spawns a dedicated **render thread** owning the `!Send` engine + GPU and blocks until init succeeds. The host then drives the engine from any thread with non-blocking command calls (`carapace_pointer`, `carapace_invalidate`, `carapace_set_frame_rate`, `carapace_release_surface`); the render thread renders into the IOSurface pool and pushes each finished frame back via the vtable's `frame_ready` callback. Hit-testing reads a published scene snapshot directly on the caller's thread. `carapace_destroy` tears it down.

## ABI version

```c
#define CARAPACE_ABI_MAJOR 3
#define CARAPACE_ABI_MINOR 2
uint32_t carapace_abi_version(void);   // returns (MAJOR << 16) | MINOR  — lib.rs:29-32
```

Additive changes bump MINOR; breaking changes bump MAJOR; a host compares this against the header constants at load time. `CarapaceStatus` is additive-only (append variants, never reorder).

## Status codes

```c
typedef enum CarapaceStatus {          // guard.rs:18-28
  CarapaceOk         = 0,
  CarapaceErrNullArg = 1,
  CarapaceErrBadSkin = 2,
  CarapaceErrGpuInit = 3,
  CarapaceErrPoisoned = 4,
  CarapaceErrPanic   = 5,
} CarapaceStatus;
```

Returned by every fallible export. On a non-`Ok` return, call `carapace_last_error` for a message.

## Error handling

```c
uintptr_t carapace_last_error(char *buf, uintptr_t cap);   // guard.rs:98-113
```

Copies the calling thread's last error into `buf` (NUL-terminated, truncated to `cap`) and returns the full length needed. Pass `buf == NULL` / `cap == 0` to query the length first, then retry. Backed by a thread-local, so it's **per-thread**, not per-handle.

## Lifecycle — create / destroy

```c
CarapaceStatus carapace_create(const CarapaceCreateDesc *desc, CarapaceEngine **out); // handle.rs:107-205
void           carapace_destroy(CarapaceEngine *ptr);                                 // handle.rs:212-223
```

- **`carapace_create`** validates the descriptor, installs a process-wide panic hook (once), spawns the `"carapace-render"` thread that builds the engine + GPU + per-surface present pool, and **blocks** on an init handshake — so it returns synchronously with `Ok`, `ErrBadSkin`, `ErrGpuInit`, or `ErrPanic`. On success `*out` receives the opaque handle; on failure `*out` stays null and the thread is joined before returning.
- **`carapace_destroy`** is null-safe: reclaims the handle, sends `Shutdown` (ignored if the thread already exited), and joins. Valid even on a poisoned handle.

### `CarapaceCreateDesc`

```c
typedef struct CarapaceCreateDesc {    // handle.rs:73-88 (#[repr(C)])
  const char          *skin_dir;       // NUL-terminated UTF-8 skin path (borrowed for the call)
  CarapaceHostVTable   vtable;         // host callbacks (fn pointers must outlive the engine)
  const IOSurfaceRef  *surfaces;       // caller-owned pool (>=1), each w×h BGRA; must outlive the engine
  uint32_t             surface_count;
  IOSurfaceRef         content_surface;// optional live host content for a view{ id="host" } cutout; null = none
  uint32_t             w, h;           // surface pixel dimensions
} CarapaceCreateDesc;
```

The render thread round-robins through `surfaces`. `content_surface`, if set, is composited into a skin's `view{}` region (re-uploaded each frame to avoid a GPU-cached-import staleness bug).

## Host callback vtable

```c
typedef struct CarapaceHostVTable {    // host.rs:11-22
  void *ctx;                                             // opaque host context, passed to every callback
  bool (*get_num)(void *ctx, const char *key, double *out);          // numeric state read
  bool (*get_str)(void *ctx, const char *key, char *buf, uintptr_t cap); // string state read (host writes buf)
  void (*invoke)(void *ctx, const char *action);        // perform a host action
  void (*frame_ready)(void *ctx, uint32_t index, uint64_t frame_id); // a frame landed in surfaces[index]
} CarapaceHostVTable;
```

"Swift is the host." `get_num`/`get_str`/`invoke` are the C mirror of the Rust [`Host`](./engine-api.md#host) trait. `frame_ready` (a v2 addition) fires **on the render thread** after each rendered frame, once its scene snapshot is published (so a reacting host never sees a stale snapshot). `frame_id` is a monotonic counter from 1.

> **`frame_ready` contract:** must be thread-safe and non-blocking, and **must not call any `carapace_*` function** — re-entering the queue/loop can deadlock.

## Driving the engine (non-blocking)

```c
CarapaceStatus carapace_pointer(CarapaceEngine *ptr, double x, double y, CarapacePointerKind kind); // handle.rs:305-326
CarapaceStatus carapace_invalidate(CarapaceEngine *ptr);                                             // handle.rs:234-243
CarapaceStatus carapace_set_frame_rate(CarapaceEngine *ptr, uint32_t fps);                           // handle.rs:252-264
CarapaceStatus carapace_release_surface(CarapaceEngine *ptr, uint32_t index);                        // handle.rs:272-284
```

All four just enqueue a command and return immediately, callable from any thread:

- **`carapace_pointer`** — pointer input in **design-canvas** coordinates; always invalidates so input shows even while paused.
- **`carapace_invalidate`** — request one frame (or wake a paused engine). Best-effort under full surface backpressure.
- **`carapace_set_frame_rate`** — `fps == 0` pauses (renders only on invalidate/input); default is 60.
- **`carapace_release_surface`** — return `surfaces[index]` to the pool once the host is done displaying it.

```c
typedef enum CarapacePointerKind { Press=0, Release=1, Move=2, Enter=3, Leave=4 } CarapacePointerKind; // handle.rs:289-295
```

Only `Press` is modeled by the engine today; the others are plumbed through as forward-compatible no-ops.

## Skin swapping

```c
CarapaceStatus carapace_swap_skin(CarapaceEngine *ptr, const char *skin_dir); // handle.rs

CarapaceStatus carapace_swap_skin_resized(CarapaceEngine *ptr,
                                          const char *skin_dir,
                                          const void *const *surfaces,
                                          uint32_t surface_count,
                                          uint32_t width,
                                          uint32_t height,
                                          const void *content_surface); // handle.rs
```

Both are **seamless**: the outgoing skin keeps rendering/animating while the incoming skin loads off the render thread, then the two crossfade per the incoming skin's `[transition]` table (default: crossfade over 250 ms; `kind = "cut"` for an instant swap) — no render-loop stall either way. Both are synchronous, blocking until the render thread has loaded the new skin and begun the transition.

- **`carapace_swap_skin`** — swaps to the skin at `skin_dir`, keeping the **same** IOSurface pool and dimensions; the engine scales the incoming skin's canvas into the existing pool. Use this when skins share a design canvas size. On a bad `skin_dir` this returns `ErrBadSkin` and leaves the current skin (and pool) running unchanged.
- **`carapace_swap_skin_resized`** — swaps to the skin at `skin_dir` **and** adopts a new host-provided surface pool sized `width`×`height` (the incoming skin's native size), so each skin can render at its own resolution instead of being scaled to a shared canvas. The incoming skin renders native-size; the outgoing skin is scaled into the new pool while it crossfades out. On `Ok`, the caller may free the **old** surfaces and must keep the **new** ones (and `content_surface`, if set) alive until the next swap or `carapace_destroy`. On a bad `skin_dir` this returns `ErrBadSkin` and leaves the current skin and pool intact — the new surfaces are not adopted.

## Hit-testing (snapshot, caller's thread)

```c
CarapaceStatus carapace_hit_test(CarapaceEngine *ptr, double x, double y, CarapaceHitKind *out); // hit.rs:36-74

typedef enum CarapaceHitKind { Passthrough=0, Control=1, Drag=2 } CarapaceHitKind;  // hit.rs:16-23
```

Classifies a point against the **latest published scene snapshot**, entirely on the caller's thread — no queue round-trip, sub-millisecond, at most one frame stale. Use it to decide passthrough vs. control vs. window-drag before/independent of sending a pointer. `Passthrough` = event falls through the skin; `Control` = skin consumes it; `Drag` = host should move the window. It carries its own `catch_unwind`: a panic here returns `ErrPanic` without poisoning the handle.

## Tier query

```c
CarapaceStatus carapace_active_tier(CarapaceEngine *ptr, CarapaceTier *out);        // handle.rs:357-376
typedef enum CarapaceTier { Readback=1, Shared=2 } CarapaceTier;                    // handle.rs:93-96
```

Reports the resolved present tier: `Shared` = zero-copy GPU path (Metal texture aliasing the IOSurface), `Readback` = CPU round-trip fallback. If any surface in the pool falls back, the reported tier is the weakest. Valid immediately after create.

## Threading & safety model

- **One render thread per handle**, spawned by `carapace_create`, joined by `carapace_destroy`. It exclusively owns all `!Send` engine/GPU state.
- **Command exports** (`pointer`/`invalidate`/`set_frame_rate`/`release_surface`) are non-blocking and callable from any thread. `carapace_create`/`carapace_destroy` are the only blocking exports.
- **`frame_ready`** fires on the render thread (see its contract above). **`carapace_hit_test`** and **`carapace_active_tier`** run on the caller's thread against the published snapshot.
- The opaque `CarapaceEngine` handle is a thread-safe front-end (command sender + snapshot cell + poison flag + join handle + tier); the real engine never crosses back.

### Panic → poison

Carapace runs inside the host process and never `abort()`s. If the render thread panics, the message is captured, the handle's `poisoned` flag is set, and the thread exits. Every subsequent command export returns `CarapaceErrPoisoned` and copies the panic message into the caller thread's `last_error` (so `carapace_last_error` retrieves it). A poisoned handle stays poisoned; `carapace_destroy` remains valid. (Exception: a panic inside `carapace_hit_test` returns `ErrPanic` and does not poison, since it never touched the render thread.)

## Minimal integration sketch

```c
CarapaceHostVTable vt = { .ctx = myctx, .get_num = my_get_num, .get_str = my_get_str,
                          .invoke = my_invoke, .frame_ready = my_frame_ready };
IOSurfaceRef pool[3] = { s0, s1, s2 };
CarapaceCreateDesc desc = { .skin_dir = "/path/to/skin", .vtable = vt,
                            .surfaces = pool, .surface_count = 3,
                            .content_surface = NULL, .w = 480, .h = 320 };
CarapaceEngine *eng = NULL;
if (carapace_create(&desc, &eng) != CarapaceOk) { /* carapace_last_error(...) */ }

// input:
carapace_pointer(eng, x, y, Press);
// in my_frame_ready(ctx, index, id): display pool[index], then later:
carapace_release_surface(eng, index);

carapace_destroy(eng);
```
